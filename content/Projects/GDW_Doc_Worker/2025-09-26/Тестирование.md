# Плагин `GDW_Convert`

## Тест-кейс 1. Обычное использование, один файл

1. Дать программе на вход папку, содержащую один Word-документ для конвертации;
2. Настроить потоки (я выбрал 1 поток);
3. Выбрать формат (далее будут все форматы, имеющиеся в программе);
4. Нажать кнопку "Старт".

P.S. Word-документ с заголовками, жирным, курсивным, подчеркнутым, зачеркнутым и выделенным цветом текстом. Так же нумерованный и ненумерованный список, локальная картинка и картинка, вставленная из интернета.

![[CleanShot-2025-09-26-at-15.24.13.png | 300]]

**Ожидаемый результат:**
 В программе есть отдельное окошко логов, оно должно в реальном времени отображать происходящие процессы.

- Программа примет на вход файл из директории;
- Счётчик `"Всего"` изменит значение на `1`;
- Создастся отдельная папка `Convert` для конвертированного файла; 
- Программа конвертирует word-документ в markdown и поместит готовый файл в папку `Convert`;
- Счётчик `Успешно` изменит значение на `1`;
- Программа закончит работу;
- Программа сообщит пользователю об удачной конвертации.

**Фактический результат:**
- Программа приняла на вход файл из директории;
- После нажатия на кнопку "Старт" в окне логов появилось название docx документа из указанной директории;
- Была создана папка `Convert`; 
- В папку `Convert` был помещён итоговый md файл;
- Счётчик `"Всего"` изменил значение на `1`;
- Далее ничего не произошло, программа не завершила свою работу. Кнопка "Стоп" так же не помогла, поможет только полный перезапуск программы.

>[!important] Pandoc
>Без предварительно установленного `Pandoc` непосредственно в систему скрипт работать НЕ будет. Установленная библиотека `Pandoc` внутри проекта (`pip install --upgrade pandoc`) не поможет, `Pandoc` должен быть установлен именно в системе.

# Плагин `GDW_CleanMD`

Содержимое тестового md файла:
```
\\\**Тест**

\\\*Тест*

\\\_Тест_

\\\[Тест]

[Тест\\\]

\\\# Тест

\\\`Тест`


\\**Тест**

\\*Тест*

\\_Тест_

\\[Тест]

[Тест\\]

\\# Тест

\\`Тест`
```

1. Дать программе на вход папку, содержащую md файл для очистки;

- Программа примет на вход файл из указанной директории;
- Программа очистит файл от символов экранирования;
- Программа завершит работу;
- Программа сообщит пользователю о успешной очистке.

**Фактический результат:**
- Программа приняла на вход файл из указанной директории;
- В окне логов вывелось: `Обработка: 1.md`;
- Файл немного почистился от символов экранирования.
